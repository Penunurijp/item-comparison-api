# Plan del proyecto

## Objetivo del proyecto
Desarrollar una API RESTful en Java 17 con Spring Boot 3.5.4 que permita comparar items de un catálogo a través de un endpoint específico con datos persistidos de un archivo JSON.

## Stack tecnológico utilizado
- **Java 17**: Lenguaje de programación.
- **Spring Boot 3.5.4**: Framework para crear aplicaciones Java.
- **Maven**: Herramienta de gestión de proyectos y dependencias.
- **Lombok**: Biblioteca para reducir el boilerplate en el código.
- **JUnit 5**: Framework para pruebas unitarias.
- **Mockito**: Biblioteca para crear mocks en pruebas unitarias.
- **JSON**: Formato de datos para persistencia y comunicación.
- **Git**: Sistema de control de versiones para el código fuente.
- **Copilot**: Asistente de IA para sugerencias de código.

## Reglas del proyecto
- No usar bases de datos, solo persistir datos en un archivo JSON.
- Manejo de errores
- Documentación del código
- Pruebas unitarias
- Uso de buenas prácticas de programación

## Estrategias de desarrollo
1. Definir modelo Producto con atributos necesarios.
2. Implementar servicio con inicializacion del JSON junto con validaciones.
3. Crear controlador REST para manejar solicitudes de comparación.
4. Implementar manejo de errores y validaciones.
5. Escribir pruebas unitarias para asegurar la funcionalidad.
6. Documentar el código y el uso de la API.
7. Realizar revisiones de código y pruebas finales.
